Q.1 Difference between == and .equals() in Java
| `==` Operator                              | `.equals()` Method                     |
| ------------------------------------------ | -------------------------------------- |
| Compares **memory references** (addresses) | Compares **content/values** of objects |
| Used with **primitive & reference types**  | Used only with **objects**             |
| For primitives, compares actual values     | Logic depends on class implementation  |
| Cannot be overridden                       | Can be overridden                      |
| Faster, checks equality at reference level | Checks logical equality                |

For ex-
String s1 = new String("Java");
String s2 = new String("Java");
System.out.println(s1 == s2);      // false
System.out.println(s1.equals(s2)); // true
Explanation-
== checks if both variables point to the same object, while .equals() checks if the contents are the same.

C, Python (standalone functions)              | Java, Python, C++ (class methods)                      |
->Hence, functions are nothing but methods in java
Code-
class Demo {
    void show() {   // method
        System.out.println("This is a method");
    }
}

public class Test {
    public static void main(String[] args) {
        Demo d = new Demo();
        d.show();   // method call
    }
